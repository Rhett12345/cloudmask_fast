# FY-3D MERSI-II 云检测算法改进规划

## 现状总结

### 已完成的修复 (v2.4.3)
- CLM bit-decode 从 MSB 恢复为 LSB（匹配 Fortran `convert_cloud_mask` 约定）
- BT108 红外增强云图加载修复（波段索引、定标路径、波数→波长转换）

|------|------|-----------|-----|------|-------|------|
| 0740 | 北极夜间 (59-82N) | 79.6% | 97.4% | 13.7% | 0.558 | 可用 |

### 已知硬件限制
- MERSI-II 缺少 13.9um CO2 切片通道（MYD35 在南极的核心依赖）
- 4.05um 在南极 BT11<200K 时完全失效（0% 有效像素）
- 所有 BTD 在南极 BT11~190K 时分离度 <0.5σ，物理信息不足

## 改进路线图

### 第一阶段：诊断工具 + 系统性评估（短期，1-2天）

**目标**：建立完整的场景级诊断能力，明确每个场景的改进空间

1. **场景分层验证脚本** (`visualize/scene_validation.py`)
   - 按地表类型 x 昼夜 x 纬度带分层统计
   - 输出每层的 Agreement / POD / FAR / HSS / 云量偏差
   - 识别最需要改进的具体场景（如：极地海洋夜间、沙漠白天等）

2. **BT/BTD 分布对比工具** (`python/compare_btd.py`)
   - 对每个场景，对比 MERSI vs MYD35 的 BT11、BTD 11-12、BTD 8-11 等分布
   - 用 MYD35 标签（cloud/clear）计算 MERSI 各测试的最优阈值
   - 输出阈值调整建议（与当前阈值的差异）

### 第二阶段：夜间阈值系统性调优（中期，3-5天）

**目标**：用 MYD35 标签重新标定所有夜间场景的阈值

3. **扩展 tune_thresholds.py 到夜间场景**
   - 已有白天调优框架 (`python/tune_thresholds.py`)，复用到夜间
   - 针对 PolarNite_land、PolarNite_ocean、LandNite、ocean_nite 四个夜间模块
   - 用多天数据（不只 20220803）统计稳定阈值

4. **重点调整的阈值**（从当前分析识别）：
   - `pno86_73`：极地海洋 8.6-7.3 BTD，当前 24K，但 SST>=280K 条件太严
   - `nl7_11s`：陆地夜间 7.3-11 BTD，上次从 -10→-26K，需要验证
   - `pnl11_12hi`：极地陆地 11-12 BTD，当前 3.0K，但在低温区物理上不成立
   - `no86_73`：海洋夜间 8.6-7.3 BTD，上次从 17→24K，需要验证

### 第三阶段：南极专项处理（中期，3-5天）

**目标**：接受 IR-only 限制，用辅助信息弥补

5. **引入 NWP 温度廓线辅助检测**
   - 南极冬季典型特征是近地表逆温层（clear sky 时地表比上空冷）
   - 比较 NWP 地表温度与 925hPa/850hPa 温度，逆温 → clear sky indicator
   - 不依赖 IR 通道，可与现有测试互补

6. **BT11 空间纹理测试增强**
   - 南极冰盖表面极其均匀（clear），云层会引入纹理
   - 增大空间变异性测试的权重（目前 uniform=false 时跳过）
   - 可考虑更大窗口（5x5 而非 3x3）检测云边缘

7. **质量标志位**
   - 对 BT11 < 230K 的极冷场景设置降低的 QA 置信度
   - 让用户知道哪些像素的云检测结果可信度低

### 第四阶段（长期，视需求）

8. **白天场景验证**：当前测试数据都是夜间（SZA fill 值），需要正常白天数据进行白天算法验证
9. **多时相一致性**：对比前后轨道同一位置的云检测结果
10. **引入 OISST 替代 NWP SST**：提高海洋场景的 SST 测试可靠性

---

## 优先级建议

1. 先做**第一阶段**（场景分层诊断），明确最大的改进空间在哪
2. 如果南极以外场景的夜间阈值有 >5% 的提升空间 → 做**第二阶段**（调优）
3. 如果南极需要从 HSS=0.03 提升到可用水平 → 必须做**第三阶段**（NWP+纹理）
4. 白天数据的验证需要先解决 SZA fill 值问题

## 验证方法

每次修改后，运行三轨道验证：
```bash
python run_fylat.py --date 20220803 --cores 3
python visualize/run_validation.py --data_dir /data/Data_yuq/fy3_cloud/20220803/ ...
```
比较修改前后的分层统计和全局 HSS，确保不退化。
